Configuration Overview

The FantDev Trading Platform uses environment variables for configuration management, supporting both Python and Bun/TypeScript runtimes. This approach provides:

  • 🔒 Security: Sensitive data stays out of version control
  • 🚀 Flexibility: Easy environment-specific configurations
  • 🔧 Simplicity: No code changes needed for deployment
  • ✅ Validation: Automatic checking of required fields
  • 🎯 Type Safety: Full TypeScript and Python type support
Quick Start: Copy .env.example to .env and update with your values to get started.

Environment Setup

Environment File Priority

Configuration files are loaded in the following order (highest priority first):

  1. .env.local - Local overrides (never committed)
  2. .env.[NODE_ENV] - Environment-specific (e.g., .env.production)
  3. .env - Base configuration

Initial Setup

# 1. Copy the example configuration
cp .env.example .env

# 2. Edit with your values
nano .env  # or use your preferred editor

# 3. For production deployment
cp .env.example .env.production
# Edit production-specific values

# 4. For local development overrides
cp .env.example .env.local
# Add your local development settings

Runtime Environment Variables

You can also set variables directly when running commands:

# Bun/Node.js
NODE_ENV=production bun run server

# Python
BOT_TOKEN=xxx python3 main_bot.py

# Multiple variables
NODE_ENV=test LOG_LEVEL=debug bun test

Rate Limiting

Variable Description Default
RATE_LIMIT_ENABLED Enable rate limiting true

RATE_LIMIT_WINDOW Time window in seconds 60
Production Security: In production, you MUST:
  • Generate cryptographically secure random keys (minimum 32 characters)
  • Use strong, unique passwords
  • Enable HTTPS/TLS for all connections
  • Rotate keys regularly
  • Never use default or example values

Generating Secure Keys

# Generate a secure JWT secret
openssl rand -base64 32

# Generate using Node.js/Bun
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Generate using Python
python3 -c "import secrets; print(secrets.token_hex(32))"

Troubleshooting

Common Issues

1. Environment Variables Not Loading

# Check if .env file exists ls -la .env # Verify file permissions chmod 644 .env # Test loading with Python python3 -c "from src.config_loader import Config; Config.print_config()" # Test loading with Bun bun run src/configLoader.ts

2. Port Already in Use

# Find process using port lsof -i :5000 # Kill process (replace PID with actual process ID) kill -9 PID # Or use different port FLASK_PORT=5001 python3 portal_server.py

3. Database Connection Issues

# Check database file permissions ls -la *.db *.json # Fix permissions chmod 644 customer_database.json chmod 644 *.db # Test database connection python3 -c "from src.config_loader import Config; print(Config.get_database_url())"

4. Missing Required Variables

# Validate configuration python3 src/config_loader.py # Check specific variable echo $BOT_TOKEN # Set variable temporarily export BOT_TOKEN="your_token_here"

Debug Mode

Enable debug mode for detailed logging:

ENABLE_DEBUG_MODE=true LOG_LEVEL=DEBUG python3 main_bot.py

Configuration Validation

Use the built-in validation to check your configuration:

# Python validation
from src.config_loader import Config

if Config.validate():
    print("✅ Configuration is valid")
else:
    print("❌ Configuration has errors")

# TypeScript validation
import { Config } from './src/configLoader';

const { isValid, errors } = Config.validate();
if (!isValid) {
    console.error('Configuration errors:', errors);
}
